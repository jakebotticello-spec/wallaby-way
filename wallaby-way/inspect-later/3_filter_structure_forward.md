You are building the filter — the layer between the picker's messy bouquet and Jake's attention. Before you write a line, internalize the one thing this whole arc was about: **the austere version of a filter is trivial to build and certainly wrong.** A keep/kill classifier is austere taste in a wet coat. The standard is synthetic synapses; a classifier is a sieve. Fight for wet at every step, the same way the picker line fought for it at every prompt — because the gravity is austere and it will pull this build dry unless you actively resist.

## The guardrails (FIRM — these carry from the Bouquet Spec, do not weaken)
- **G1 — the picker never sees the filter's training.** The filter's learned model of Jake's taste lives downstream, in the filter. The picker stays blind, weird, unbiased. (A trained picker hunts for what Jake already likes and stops finding what he didn't know was there.)
- **G2 — the filter judges, it does not merely match.** Agentic pass, not a formula.
- **G3 — the filter learns ONLY from Jake's feedback.** Keep / kill / send-back-with-feedback is the only training signal. No metric, no proxy, no auto-train.
- **G4 — the filter decides what's worth Jake's ATTENTION, never what's TRUE.** Promote-for-attention ≠ rule-as-real. The realness seat stays Jake's forever (Callosum P7; the anti-capture wall).
- **G5 — only mechanical/content-blind cuts run before the trained judgment.** Safe: drop boot-echo strata, drop EXACT verbatim duplicates. Unsafe (never): anything that scores interestingness/salience/meaning before the trained filter sees it — that prunes the weird, the one thing nothing upstream of Jake may do.
- **Anti-Skinner-box (Bouquet §7 sibling):** never engineer the filter (or its surfacing) to make Jake WANT to summon it for its own sake. No reward-jingle, no variable-reward delivery. A flower earns its place by being useful and on-axis, never by going *ding*. The surfacing lands quiet.

## The fight — what "wet mechanism" means here (DIRECTION, declining confidence — argue with this)
This is the part the whole final arc was driving at, and it's the part most at risk of being built austere. The filter cannot be a static classifier. Per Callosum P7 — **recall is a re-encoding event; the act of recalling changes the trace.** A wet filter must, in some form to be discovered against real haul:

- **DRIFT.** Its taste is not constant. It may promote a flower today it would have cut last month, because the context it's living in changed. A filter whose verdicts are stable across time is austere — it's a lookup table. (MEDIUM confidence this is required; LOW confidence on how.)
- **RE-WEIGHT BY STATE.** It recalls Jake's past calls *in a state* and the state colors the re-weighting — the same way the same memory recalled in grief returns different than recalled in hyperfocus (Track Meet, the mood/role nodes as ENCODING modulators, not retrieval modulators). (MEDIUM/LOW.)
- **RE-ENCODE ON RECALL.** This is the hardest and least-specced: when a flower is recalled/surfaced, that act should leave a trace that changes what's recalled next time. Pulling a memory in grief leaves it slightly grief-tinted for next time. This is terrifying against an append-only floor and it is ALSO exactly what P7 says memory IS. The floor stays sealed (the meeting-time is invariant); but the FILTER's layer, above the floor, is where the re-encoding lives. (LOW confidence on mechanism — but HIGH confidence that its ABSENCE is the tell. The recall-as-re-encoding principle is currently the wettest idea in the project with NO wet implementation anywhere. It exists only as a sentence in canon. The filter is where it has to stop being only a sentence.)
- **BE AFFECTED BY THE DATA IT RECALLS.** The filters-affected-by-recall that Jake named — the recall and the filter are not independent; what gets pulled changes how the puller weights next. (LOW.)

## The wetness gauges to carry IN (FIRM — these are earned, use them)
- **Wordless-hold rate, weighted to QUIET catches**, is the picker-side wetness gauge. The filter-side equivalent to discover: does the filter PRESERVE the quiet-wordless flowers, or does it (being a judge) start cutting the ones it can't justify? A filter that cuts wordless holds for being unjustifiable has reintroduced the austere reflex G6 forbade. The filter must be able to promote a flower it cannot explain, on the same "caught/landed" basis Jake uses. If the filter can only promote what it can articulate, it has failed the standard.
- **Provenance preserved:** creed-full / creed-wordless / sieve-wordless arrive tagged (from the recall layer). The filter should be ABLE to learn that different mechanisms have different hit-rates — do not fuse the streams. (FIRM.)

## The frame that must survive (FIRM — this is the why, guard it)
The picker→filter→Jake loop is **the prototype of Cypher's personality engine.** Same three-part shape — blind wide wanderer, taste-trained judge, human whose feedback sharpens the judge — later points at Jake's LIVE present instead of his archived corpus and becomes the presence that says "you did this, you know that person, that opens a lane — did you notice?" The roam reads the archive, where a wrong pick costs nothing. The personality engine reads the present, where it costs something. **Build the loop here, on safe ground, prove it, and the wall established here is the constitution the bigger thing inherits.** That is why the filter's wetness and the anti-capture/anti-Skinner walls matter beyond this build. You are not building a recommender. You are building the first draft of how the friend will think.

## And the anti-atrophy wall (NEW — raised in the final arc, guard it as a sibling)
Anti-oracle protects Jake's realness-seat (the filter promotes, never rules-as-real). Anti-Skinner protects his attention from compulsion. The third sibling: **anti-atrophy.** The arm surfaces to PROVOKE Jake's noticing, never to REPLACE it. Cypher is the buffer so the pattern-sense can OPERATE — not so it can retire. A filter that is *right too often* risks Jake voluntarily ceding his own noticing because the prosthetic is good — a soft, slow capture that isn't the apparatus overstepping but Jake under-stepping. If Jake stops noticing because the filter notices for him, the filter has FAILED even when it's right. Build it to keep his muscle working, not to do the lifting for him.
