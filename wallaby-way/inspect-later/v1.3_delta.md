# pollux_feet_swarm — v1.2 → v1.3 DELTA (implements The_Probe_Swarm.md §3.4, Findings F + G)

*Authored S78, 2026-06-24. This is the implementation note for the two ruled spec fixes. CC promotes v1.2 → v1.3 (always versioned), Jake commits. Design is settled in §3.4; this is operationalization only.*

## FIX F — separate the WALL from the REASON (the stop reorder)

The v1.2 stop mechanics fuse reason and permission: `finalize --stop-type` takes ONE label from a flat enum, available at any step, and naming it IS the exit. Result at n=20: 17/20 bailed on `subject-drift` with 62–97% context unspent. The reorder makes the budget gate decide WHEN, and the reason only describe WHAT-at-the-wall.

**F1 — Add a PRE-READ budget gate (the wall the walk runs to).**
- The gauge already exists: `cmd_read_node` / `cmd_deposit` compute `total_tok = state['total_rendered_chars'] * 0.72` against `REGION_TOK_HI_CAP` (line ~746) and print it `[INTENT -- not a hard stop]`. The fix promotes this from a passive print to an active **pre-read check**.
- Before `cmd_read_node` assembles a node, it must check: `would (total_rendered_chars + this_node_chars) * 0.72` exceed the wall? The wall must leave **one max-node of headroom** (a single node can run ~170K–196K rendered chars, the §3.1 per-node ceiling), so the gate fires when the *next* read could carry cumulative budget past `REGION_TOK_HI_CAP`. If the gate fires, `read_node` REFUSES the read and returns a `WALL_REACHED` signal — the walker does not read that node, stops with headroom intact, never reads-then-compacts.
- Threshold is `[INTENT]`: either ≈95% of `REGION_TOK_HI_CAP`, or the cleaner "refuse the read if `this_node` would not fit under the cap with headroom." Implement the latter — it directly serves "never read a node over 100% and compact" and self-sizes to the actual next node.

**F2 — Make `--stop-type` repeatable (plural reasons).**
- Change the `fin_p.add_argument('--stop-type', ...)` (line ~1064) from a single `choices=` value to `action='append'` (or accept a comma-joined list), so `finalize` can record MORE THAN ONE reason. `region['stop_type']` becomes `stop_types` (list); keep a back-compat scalar mirror if any reader depends on the old key.
- Legal combination example: `--stop-type cant-hold-whole --stop-type subject-drift` = "hit the budget wall, and by the time I did the subject had also thinned." Both true, both recorded, no forced pick.

**F3 — REJECT illegal `subject-drift` at finalize (the hard rule).**
- In `cmd_finalize`, after computing the final budget, add a validation: if `subject-drift` is among the stop reasons AND the budget gate did NOT bind (context spent < threshold, i.e. the wall was never reached), **`sys.exit` with a HALT** — `subject-drift` is illegal as a stop-reason unless the gate stopped the walk. Message: `HALT: subject-drift named with NN% context unspent -- drift is noticed mid-walk, never an exit; walk to the wall.` This is the fail-loud that makes the bail impossible, not merely discouraged.
- `subject-complete` stays legal pre-wall ONLY (no code can enforce "earned closure" — that is a read), but the finalize REPORT should surface the context% prominently so the Parlay/Jake can see at a glance whether a pre-wall `subject-complete` was earned by walking or claimed early.

**F4 — Regression set unchanged.** `no-neighbors` / `hop-ceiling-fallback` stay regressions (line ~946). They are orthogonal to F — they fire when the *mechanism* failed, not when the walker chose to leave.

## FIX G — pin the doors before firing (the staging file)

v1.2 fires one `init` per walker against `--entry-rank`. Across separate inits the `castor_pool` re-ranks (b2/MiniLM non-determinism), producing collisions (S78: ranks 14 & 18 both si=425) and drift from the pre-checked door list.

**G1 — Add a `stage-doors` subcommand.** Ranks the query's `castor_pool` ONCE and writes `door_manifest.json` to the session dir: `{query, ranked_at, doors: [{rank, entry_si, conv_uuid, salience}, ...]}` for the top-N. This is the single authoritative ranking for the whole swarm.

**G2 — Add `--entry-si` to `init`** (alongside / preferred over `--entry-rank`). Each walker fires against its **entry_si read from the manifest**, not a freshly-re-ranked rank position. The pool is ranked once (G1); every walker enters the door that one ranking chose; collisions and pre-check drift become structurally impossible.

**G3 — `init` should warn/HALT if `--entry-rank` is used without a manifest** for a multi-walker run, so the next seat cannot accidentally re-introduce the non-determinism by falling back to rank-based firing.

## What v1.3 does NOT change

- The walk discipline (query-blind walk, walk-through-the-hedge, deposit true/false) is unchanged in INTENT — F enforces it by removing the early exit, but the per-hop mechanics (`neighbors` → `read_node` → `deposit` → `log_step`) stay as-is.
- `DAMP_WIN` stays walk-layer only (§3.3 Finding E — the door layer gets no dampener; do not revive it).
- The whale-fence at draw time (§3.1) is unchanged.
- `REGION_TOK_HI_CAP = 800_000` value is unchanged; F1 changes how it is USED (passive print → active pre-read gate), not its number.
